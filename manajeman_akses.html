Manajemen User dan Hak Akses Database di MySQL

A. Membuat User Akses

Secara default, jika pertama kali menginstall MySQL maka hanya ada 1 user yaitu ‘root’. User root merupakan pemilik hak tertinggi dalam MySQL. Saran saya jangan menggunakan user root untuk mengoneksikan aplikasi database anda ke MySQL demi Keamanan. Secara umum query untuk membuat user baru adalah:
“ CREATE USER 'namauser'@'host' IDENTIFIED BY 'password'; ”
contoh 1:
dari contoh di atas maka si user budi ini hanya bisa melakukan akses ke MySQL dari host dengan IP 192.168.56.11.
contoh 2:

Dari contoh di atas maka si user rini hanyak bisa melakukan koneksi dari host yang sama dengan host MySQLnya. Hal ini cocok dilakukan jika server MySQL dan databasesnysa berada di server yang sama.
contoh 3:

Dari contoh di atas maka si user dinda bisa melakukan koneksi dari host manapun atau tidak terbatas ip/host nya. Meskipun dengan % (wild card) host ini tampak fleksibel dalam hal konektivtias, namun juga beresiko karena si user ini bisa akses dari host manapun.

B. Menghapus User

Query untuk menghapus user yang sudah dibuat, printahnya adalah:
“ DROP USER ‘namauser’@’host’; “
contoh:

Printah diatas digunakan untuk menghapus user rini yang mengakses dari localhost. Dan query ini juga bisa digunakan untuk menghapus host yang menggunakan ip dan % (wild card).

C. Memberikan Hak Akses User ke Database Tertentu

Memberi hak akses user ke database tertentu berfungsi supaya user tersebut hanya bisa mengakses databases yang diberikan hak akses dan juga hanya bisa melakukan query tertentu Query SQL secara umum untuk pembatasan hak akses user adalah:
“ GRANT hak1, hak2, ….. on namadatabases.namatabel TO ‘namauser’@’hos’; “
Beberapa macam hak akses yang bisa diberikan kepada user adalah

  1. SELECT: si user bisa melakukan perintah SELECT
  2. INSERT: si user bisa melakukan perintah INSERT untuk menambahrecord
  3. UPDATE: si user bisa melakukan perintah UPDATE untuk update record
  4. DELETE: si user bisa melakukan perintah DELETE untuk hapus record
  5. CREATE: si user bisa melakukan perintah CREATE untuk membuat tabel baru
  6. ALTER: si user bisa melakukan perintah ALTER untuk mengubah struktur tabel
  7. DROP: si user bisa melakukan perintah DROP untuk menghapus tabel

Contoh 1:

Contoh di atas berarti memberikan hak akses ke user budi yang mengakses dengan host 192.168.56.11 hak akses yang diberikan hanyal select ke database coba dan semua table yang ada didalamnya dimana user hanya bisa menjalankan query select di database coba.
Contoh2:
Dari printah diatas kita juga bisa memberikan hak akses lebih dari 1 ke user

Di contoh di atas saya memberikan hak akses insert dan delete ke user budi tapi hanya di database coba saja dan di semua table yang. Tapi jika anda mau lebih spesifik anda bisa mencantum kan nama table nya sekaligus
Contoh3:

Maksud dari query diatas, adalah memberikan hak akases pada user budi yang mengakases dari host 192.168.56.11 ke database coba. User ini mendapatkan full akses ke databases coba sehingga bisa melakukan semua query
Contoh4:

Maksud dari query di atas adalah memberikan hak akses pada user budi yang mengakses dari host 192.168.56.11 ke semua databases yang ada dengan full akses.
Note: setiap kita memberikan printah GRANT jangan lupa meberikan perintah query.
“FLUSH PIVILEGES;“
Query di atas untuk mereload semua privileges di MySQL. Atau bisa juga dengan melakukan restart sevice MySQL

D. Mencabut Hak Akses User

Query untuk mencabut hak akses user secara umum adalah:
“REMOKE hak1, hak2, … ON namadatabase.namatable FROM ‘namauser’@’host’ “
contoh1:

Perintah diatas bertujuan untuk mencabut hak akses unutk menjalankan printah SELECT di database coba pada semua tabel dari user budi yang mengakses dari host ‘192.168.56.11’
Contoh2:

Perintah di atas bertujuan untuk mencabut hak untuk menjalankan perintah SELECT dan DELETE di database cobap ada semua tabel, dari user budi yang mengakses dari host 192.168.56.11.
contoh3:

Perintah diatas bertujuan untuk mencabut semua hak akses di di semua database pada semua tabel, dari user budi yang mengakses dari host192.168.56.11’
